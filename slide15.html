Goroutine Pool

Kiểm soát Concurrency với Worker Pool

Concurrency Architecture

Bài toán

  • Hệ thống có rất nhiều job đẩy về liên tục.
  • Không thể spawn vô hạn goroutine (gây tràn RAM, sập HĐH).
  • Cần giới hạn concurrency (đồng thời).
  • Phải tránh Overload hệ thống downstream như DB, 3rd API.

Giải pháp

Goroutine Worker Pool
Tạo sẵn N worker cố định và phân phối job cho chúng thông qua Channel.

Key Benefits

  • Giới hạn số lượng Goroutine chạy đồng thời.
  • Tận dụng sức mạnh CPU cực kỳ hiệu quả.
  • Tránh Overload DB / API (rate limit tự nhiên).
  • Code tường minh, dễ dàng testing và maintain.

Code minh họa 

jobs := make(chan int, 100)

// Khởi tạo worker logic
worker := func(id int) {
    for job := range jobs {
        fmt.Printf("Worker %d processing job %d\n", id, job)
    }
}

// Tạo Worker Pool (Bật 3 worker chạy nềnn)
for i := 0; i < 3; i++ {
    go worker(i)
}

// Bắn 10 Jobs vào Queue
for i := 1; i <= 10; i++ {
    jobs <- i
}
close(jobs)
