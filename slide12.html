DEMO: Cơ chế Aggregating

DNS Log → Materialized View → AggregatingMergeTree

1.1 Raw table (log DNS)

CREATE TABLE dns_log (
    timestamp DateTime,
    client_ip String,
    domain String,
    rcode UInt8
)
ENGINE = MergeTree
ORDER BY (domain, timestamp);
👉 Đây là table ingest log (append-only)

1.3 Materialized View

CREATE MATERIALIZED VIEW mv_dns_agg
TO dns_agg AS
SELECT
    domain,
    countState() AS total_query,
    countIfState(rcode != 0) AS error_query
FROM dns_log GROUP BY domain;
👉 Mỗi lần insert raw → MV kích hoạt → insert state

1.2 Aggregation Table (Đích)

CREATE TABLE dns_agg (
    domain String,
    total_query AggregateFunction(count),
    error_query AggregateFunction(countIf, UInt8)
)
ENGINE = AggregatingMergeTree()
ORDER BY domain;
👉 Lưu state dạng nhị phân, không phải giá trị (value)

Mục tiêu

  • Đếm số query theo domain.
  • Đếm số lỗi (rcode != 0).
  • Xây dựng Dashboard Realtime (Query < 100ms)

1.4 Query Dashboard

SELECT
    domain,
    countMerge(total_query) AS total,
    countIfMerge(error_query) AS errors
FROM dns_agg
GROUP BY domain ORDER BY total DESC LIMIT 10;

1.5 Chuyện gì xảy ra phía sau?

Insert log → MV transform → Lưu State → Background Merge → Query
