2. Core Engines & Hiệu Suất Xử Lý

Các Engine quan trọng

  • MergeTree (core storage engine)
  • ReplacingMergeTree (xóa/dedup dữ liệu)
  • SummingMergeTree (tự động tính tổng)
  • AggregatingMergeTree (pre-aggregation)
  • Distributed (query cluster)
Không cần nhớ hết, nhưng hiểu concept.

Compression (Nén)

  • Dữ liệu đứng cạnh cùng kiểu dữ liệu compress rất tốt.
  • Giảm thiểu kích thước lưu trữ (10x - 100x).
  • Giảm tải rõ rệt Disk I/O.
  • Tăng tốc độ trả về của Query.
CPU giải nén (decompress) nhanh hơn đọc disk.

Vectorized Execution

  • Xử lý theo mảng/batch (không duyệt từng dòng).
  • Tận dụng sức mạnh đa luồng CPU (Tập lệnh SIMD).
  • Giảm tối đa call overhead.
  • Tăng tốc cực lớn cho Aggregation.
Đặc biệt quan trọng với các Query chứa GROUP BY.

Insert Performance

  • Ghi Append-only (không hề có lock state).
  • Batch insert cực kỳ nhanh (không có độ trễ).
  • Vô cùng phù hợp với luồng Streaming / Data Log.
  • Chịu tải ghi Insert hàng triệu records mỗi giây.
CHÚ Ý: Không phù hợp việc insert lắt nhắt từng dòng nhỏ lẻ.

Distributed & Scale

  • Query xuất phát sẽ được phân tán song song qua nhiều Nodes.
  • Mỗi Node tự xử lý 1 phần khối lượng dữ liệu độc lập.
  • Kết quả sau cùng được Merge tổng lại để trả về cho Client.
  • Cơ chế Scale ngang (Horizontal block) rẽ nhánh vô cùng dễ dàng.
Distributed Table thực chất chỉ là Layer Logical (không chứa data cứng).
