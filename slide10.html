Xử Lý Lỗi (Error Handling)

Sự đối lập triết lý giữa Golang và Python

Explicit Error Handling

data, err := fetch() if err != nil { return err } process(data)

Error được coi là một biến/giá trị bình thường (Value).

Phải kiểm tra lỗi (check) một cách rõ ràng (Explicit).

Không tồn tại khái niệm try/catch.

👉 Triết lý: "Errors are values"

Exception Handling

try: data = fetch() process(data) except Exception as e: print(e)

Exception tự động ném (propagate) lên Call stack.

Phải bọc bằng try/catch block.

Flow không hiển thị rõ hàm nào sẽ quăng lỗi.

👉 Triết lý: "Easier to ask forgiveness than permission"

So sánh nhanh

Tiêu chí Golang Python
Cách xử lý Explicit (Giá trị trả về) Exception (Ngoại lệ)
Flow luồng đi Rõ ràng (Đọc từ trên xuống) Ẩn (Nhảy ra khỏi block)
Độ khó Debug Dễ (Biết ngay hàm nào lỗi) Khó hơn (Truy stacktrace)
Độ dài Code Dài, lặp lại nhiều `if err != nil` Ngắn, bọc 1 lần
Runtime Cost Thấp Exception đắt đỏ
Lời Khuyên (Insight)
Go: Kiểm soát lỗi cực kỳ rõ ràng, predictable, không bất ngờ.
Python: Code gọn gàng, nhưng rất dễ vô tình "nuốt lỗi" nếu bọc khối Try quá bự.
Hàm hố cần tránh (Pitfall)
Py: except Exception: pass ❌ mất lỗi
Go: _ = err ❌ ignore error
Go Best Practices
  • Luôn check err: Không được bỏ qua `err`.
  • Wrap error để trace: return fmt.Errorf("fail: %w", err) để giữ lại gốc lỗi.
  • Không dùng Panic: Panic chỉ dùng khi app chết (crash), tuyệt đối không dùng cho Business Logic.
