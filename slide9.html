Go Channel & Worker Pool

Ví dụ thực chiến: Phân phối tải bằng cơ chế hàng đợi

Giao tiếp Channel

Channel là cái tẩu (đường ống) kết nối các Goroutine để truyền dữ liệu một cách an toàn.

1. Đẩy dữ liệu (Send) ch <- ev
Bắn biến ev vào bên trong channel ch
2. Lấy dữ liệu (Receive) ev := <-ch
Hứng dữ liệu từ ch đổ ra ngoài gán vào biến ev.

Giải mã luồng Code

  • Buffered Channel: make(chan int, 5) tạo đường ống có độ dài 5. Giúp Producer bắn nhanh hơn mà không bị block ngay lập tức.
  • Fan-out: Vòng lặp go worker() nhân bản 3 tiến trình chờ sẵn ở cổng Channel.
  • Close & Range: Vòng lặp range jobs bên trong worker tự động hút cạn channel cho tới khi Master kích hoạt lệnh close(jobs).
package main

import (
    "fmt"
    "time"
)

func worker(id int, jobs <-chan int) {
    for job := range jobs {
        fmt.Printf("Worker %d xử lý job %d\n", id, job)
        time.Sleep(1 * time.Second)
    }
}

func main() {
    // Tạo Buffered Channel có sức chứa 5 slot
    jobs := make(chan int, 5)

    // Spawn 3 tiến trình worker ngậm sẵn channel này
    for i := 0; i < 3; i++ {
        go worker(i, jobs)
    }

    // Đẩy dữ liệu (Push) liên tục 10 jobs vào channel
    for i := 1; i <= 10; i++ {
        jobs <- i  // Send operation
    }

    // Báo hiệu không còn job nào được gửi nữa
    close(jobs)

    // Đợi các worker hoàn tất (giải pháp demo, thực tế nên dùng WaitGroup)
    time.Sleep(5 * time.Second)
}
