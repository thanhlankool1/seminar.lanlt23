Error Handling Thực Chiến

Ví dụ cụ thể: Bắt và phân loại lỗi xuyên suốt 3 Layer ứng dụng

1
Tầng DB (Low-level)
Tương tác trực tiếp với Database. Lỗi sinh ra ở đây thường là lỗi hệ thống lõi (VD: chuỗi trống, mất kết nối, v.v).
func getUser(id string) (User, error) {
    if id == "" {
        return User{}, sql.ErrNoRows
    }
    
    /* ... logic query DB ... */

    return user, nil
}
2
Map sang Error (Service)
Tầng trung gian quan trọng. Che giấu mã lỗi rò rỉ của hệ thống, chuyển đổi (Map) thành mã lỗi nghiệp vụ của riêng bạn.
var ErrUserNotFound = errors.New("user not found")

func getUserService(id string) (User, error) {
    user, err := getUser(id)
    if err != nil {
        // Kiểm tra mã lỗi lõi bằng errors.Is
        if errors.Is(err, sql.ErrNoRows) {
            return User{}, ErrUserNotFound
        }
        // Các lỗi khác thì ném lên bth
        return User{}, err
    }
    return user, nil
}
3
Xử lý ở Layer ngoài
Tầng Controller/API. Bắt mã lỗi nghiệp vụ để render ra thông điệp trả về cho người dùng (JSON/HTTP status code).
user, err := getUserService("123")

switch {
    
// Lỗi cụ thể: User = rỗng
case errors.Is(err, ErrUserNotFound):
    fmt.Println("handle not found")
    // => Return HTTP 404

// Mọi lỗi khác
case err != nil:
    fmt.Println("internal error")
    // => Return HTTP 500

}
